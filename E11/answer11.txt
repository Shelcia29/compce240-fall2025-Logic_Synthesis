A small plan of your block in file

1. Brief textual description of your processes

The FSM process controls the I²C communication sequence between the master and the slave device. 
It starts in the START state, where it generates the I²C start condition by pulling the SDA line low while SCL is high. 
Then, it moves to the SEND_BYTE state to transmit each bit of a data byte over the bus. 
After sending a byte, it transitions to the WAIT_ACK state, where it releases the SDA line and waits for an acknowledgment (ACK) from the slave. 
If an ACK is received, it proceeds to send the next byte or the next register value. If no ACK is received, it sets a retry flag. 
Finally, the FSM enters the STOP_COND state, where it generates the stop condition (SDA high while SCL high). 
Depending on whether a retry is needed or all parameters are configured, it either restarts communication or finishes the configuration by asserting the finished_out signal.
